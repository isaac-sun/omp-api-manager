# OMP API Manager v0.2.1

> Language: **English** | [简体中文](../zh-CN/releases/v0.2.1.md)

Released 2026-07-11.

## Launch crash hotfix

This release replaces v0.2.0, whose downloadable app could crash immediately after launch. The icon startup path invoked SwiftPM's generated `Bundle.module` accessor, but the custom macOS app package stores its resources in the standard `Contents/Resources` directory. The unavailable SwiftPM bundle path triggered a Swift assertion during application startup.

v0.2.1 loads the packaged icon from `Bundle.main`, keeps `Bundle.module` only for SwiftPM development runs, and adds a launch smoke test to the DMG packaging process so this class of failure is caught before publication.

The provider editor also fixes the API base URL and API key rows. Their nested form labels previously compressed the real input control to a nearly invisible caret; both fields now have explicit, full-width editable controls. Provider and model forms also use muted examples, field-level guidance, and row dividers so it is clear where to type and what each value means.

## Installation

Delete the v0.2.0 app, download `OMP-API-Manager-0.2.1-macos-arm64.dmg` and its `.sha256` file from the release assets, then drag the replacement app to Applications.

The build supports Apple Silicon and requires macOS 14 Sonoma or later. It is ad-hoc signed but not Developer ID notarized, so macOS may show an unidentified-developer warning.

## Validation

The packaged application passes strict code-signature and bundle checks, starts successfully from the finished `.app`, and the mounted DMG is checked again before release.
