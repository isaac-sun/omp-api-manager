# OMP API Manager v0.2.0

> Language: **English** | [简体中文](../zh-CN/releases/v0.2.0.md)

Released 2026-07-11.

## Provider workflow update

This release makes personalized OMP provider configuration substantially easier. It adds secure New API connection import, a richer multi-model form, selectable OMP API modes, and a fast way to duplicate an existing provider without duplicating its credentials.

## Highlights

- Import a `newapi_channel_conn` JSON object as a custom OpenAI-compatible provider. A root endpoint is normalized to `/v1`, and users can choose **Import Draft** or **Import and Apply**.
- Configure multiple models for one provider, including `id`, `name`, `contextWindow`, `maxTokens`, text/image input capabilities, reasoning support, and `cost.input`, `cost.output`, `cost.cacheRead`, and `cost.cacheWrite`.
- Enter prices as ordinary decimals or scientific notation such as `5e-1`, and use quick presets for common token limits.
- Select the OMP API mode required by the endpoint: `openai-completions`, `openai-responses`, `openai-codex-responses`, `azure-openai-responses`, or `anthropic-messages`.
- Duplicate a provider's endpoint and model configuration as a starting point for a new provider. Credentials are intentionally not copied.
- Show or hide the API key while editing a provider, while keeping saved credentials in macOS Keychain.

## Security notes

The New API connection is parsed only in memory. Its API key is written directly to macOS Keychain, while the pasted JSON is never persisted in provider metadata, diagnostics, or logs. Provider duplication also excludes API keys, so every duplicate must be given its own credential.

As before, the app does not persist prompts, responses, authorization headers, or API keys in usage records. See [Security](../../SECURITY.md) and [Privacy](../../PRIVACY.md).

## Installation

Download `OMP-API-Manager-0.2.0-macos-arm64.dmg` and its `.sha256` file from the release assets. Verify the checksum, open the DMG, and drag the app to Applications.

Place both downloaded files in the same directory, then run `shasum -a 256 -c OMP-API-Manager-0.2.0-macos-arm64.dmg.sha256`.

The build supports Apple Silicon and requires macOS 14 Sonoma or later. It is ad-hoc signed but not Developer ID notarized, so macOS may show an unidentified-developer warning. Only open a build downloaded from this repository's official release page.

You can also clone the `v0.2.0` tag, open `Package.swift` in Xcode, and run the `OMPAPIManager` scheme; alternatively run `swift run OMPAPIManager` from Terminal.

## Validation

The release source passes a strict Swift build with warnings treated as errors and all 24 automated tests.

## Reproducible packaging

On an Apple Silicon Mac with Xcode installed, run `scripts/package-dmg.sh 0.2.0`. The command creates a DMG and portable SHA-256 checksum file in `dist/`.
