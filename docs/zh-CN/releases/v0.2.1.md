# OMP API Manager v0.2.1

> 语言： [English](../../releases/v0.2.1.md) | **简体中文**

发布于 2026-07-11。

## 启动闪退热修复

此版本替换 v0.2.0；该版本的下载版应用可能在启动后立即闪退。启动时的图标读取路径调用了 SwiftPM 自动生成的 `Bundle.module` 访问器，但自定义 macOS 应用包会把资源放在标准的 `Contents/Resources` 目录。不可用的 SwiftPM Bundle 路径会在应用启动期间触发断言。

v0.2.1 在发布版中通过 `Bundle.main` 读取图标，只在 SwiftPM 开发运行时使用 `Bundle.module`，并在 DMG 打包流程中增加真实启动冒烟测试，确保此类问题在发布前被发现。

服务商编辑器也修复了 API 基础 URL 和 API Key 行。此前嵌套的表单标签会把真正的输入区域压缩成几乎不可见的光标；现在两个字段都有明确、完整宽度且可以正常编辑的输入控件。服务商和模型表单也采用淡色示例、字段级说明与行间分隔，让用户能一眼看出输入位置及每个值的含义。

## 安装

请删除 v0.2.0 应用，从 Release 资产下载 `OMP-API-Manager-0.2.1-macos-arm64.dmg` 及其 `.sha256` 文件，然后将替换后的应用拖入 Applications。

该构建支持 Apple Silicon，需要 macOS 14 Sonoma 或更高版本。它使用 ad-hoc 签名，尚未进行 Developer ID 公证，因此 macOS 可能提示未知开发者。

## 验证

打包后的应用已通过严格的代码签名和 Bundle 检查，可以从完成的 `.app` 正常启动，并会在发布前再次检查挂载后的 DMG。
