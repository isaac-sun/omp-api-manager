# OMP API Manager v0.2.0

> 语言： [English](../../releases/v0.2.0.md) | **简体中文**

发布于 2026-07-11。

## 服务商配置流程升级

这个版本让个性化 OMP 服务商配置变得更加简单：新增安全的 New API 连接导入、更完整的多模型表单、可选择的 OMP API 模式，以及无需复制凭据即可快速复制现有服务商的功能。

## 主要更新

- 将 `newapi_channel_conn` JSON 对象导入为自定义 OpenAI 兼容服务商。根端点会自动规范化为 `/v1`，并可选择“仅导入草稿”或“导入并应用”。
- 为一个服务商配置多个模型，包括 `id`、`name`、`contextWindow`、`maxTokens`、文本/图像输入能力、推理能力，以及 `cost.input`、`cost.output`、`cost.cacheRead` 和 `cost.cacheWrite`。
- 价格支持普通小数和 `5e-1` 这样的科学计数法，并提供常用 Token 限制快速预设。
- 根据端点选择所需的 OMP API 模式：`openai-completions`、`openai-responses`、`openai-codex-responses`、`azure-openai-responses` 或 `anthropic-messages`。
- 复制服务商的端点和模型配置，以此为起点创建新的服务商；凭据不会被复制。
- 编辑服务商时可显示或隐藏 API Key；已保存的凭据仍保留在 macOS Keychain 中。

## 安全说明

New API 连接仅在内存中解析。API Key 会直接写入 macOS Keychain，粘贴的 JSON 不会持久化到服务商元数据、诊断信息或日志。服务商复制同样会排除 API Key，因此每个副本都需要单独设置凭据。

与之前版本相同，应用不会在用量记录中持久化提示词、响应、授权头或 API Key。详见[安全政策](../../../SECURITY.zh-CN.md)和[隐私政策](../../../PRIVACY.zh-CN.md)。

## 安装

从 Release 资产下载 `OMP-API-Manager-0.2.0-macos-arm64.dmg` 及其 `.sha256` 文件。校验 SHA-256 后打开 DMG，并将应用拖入 Applications。

将两个下载文件放在同一目录，然后运行 `shasum -a 256 -c OMP-API-Manager-0.2.0-macos-arm64.dmg.sha256`。

该构建支持 Apple Silicon，需要 macOS 14 Sonoma 或更高版本。它使用 ad-hoc 签名，尚未进行 Developer ID 公证，因此 macOS 可能提示未知开发者。请只打开从本仓库官方 Release 页面下载的构建。

也可以克隆 `v0.2.0` 标签，在 Xcode 中打开 `Package.swift` 并运行 `OMPAPIManager` Scheme；或者在终端运行 `swift run OMPAPIManager`。

## 验证

该版本源代码已通过将警告视为错误的严格 Swift 构建，以及全部 24 项自动化测试。

## 可复现打包

在装有 Xcode 的 Apple Silicon Mac 上运行 `scripts/package-dmg.sh 0.2.0`。该命令会在 `dist/` 中创建 DMG 和可移植的 SHA-256 校验文件。
