# 网关设计

> 语言： [English](../gateway.md) | **简体中文**

状态：MVP 网关已实现。应用可以在仅回环的监听器上启动一个由用户明确选择的服务商，并实时转发普通响应或 SSE 字节块；自动重连 OMP 仍是后续工作。

网关只绑定 `127.0.0.1`，除非用户指定，否则选择未占用端口。每个入站请求都使用独立生成、存储在 Keychain 中的 Bearer Token 认证。网关从 Keychain 获取上游密钥，并将方法、路径、安全请求头和正文转发给一个选定的上游服务商。响应头和响应字节（包括 SSE 块）不会持久化。它只向 SQLite 写入脱敏的耗时、状态、服务商/模型标识和服务商报告的 Token 元数据；提示词、响应正文、API Key 和授权头绝不保存。

网关会区分服务商报告的 Token 值和不可用的值。它绝不能绑定到 `0.0.0.0`、将上游凭据复用于本地凭据，或记录 `Authorization` 请求头。
