# 配置安全

> 语言： [English](../configuration-safety.md) | **简体中文**

每一项受支持的配置写入都会遵循以下事务：

1. 读取目标文件并计算指纹（SHA-256、字节数、修改时间）。
2. 将 YAML 解析为语义值树。
3. 仅在 OMP 版本适配器接受该 Schema 后，执行目标映射更新。
4. 在变更前立即再次检查指纹。
5. 在原文件旁创建带时间戳的备份。
6. 序列化为同级临时文件，并再次解析临时内容。
7. 原子替换前再次检查源文件指纹，再重新读取确认。

若外部编辑改变了指纹，操作会以冲突失败，而不会覆盖数据。若解析、备份或替换失败，原文件保持不变。恢复备份时会验证 YAML 并使用同一事务，同时先备份当前配置。备份按最新优先列出，自动保留不会少于十个文件。

Yams 会保留已解码值树中未知字段的语义，但不能完全保留注释和格式。应用会提示这一限制，用户可以比较或恢复生成的备份。应用绝不会用字符串替换或正则表达式编辑 YAML。

高级 `models.yml` 编辑器只向 UI 返回经过脱敏的序列化内容。名称表明其为密钥的字段（`apiKey`、`authorization`、`token`、`secret` 或 `password`）会显示为脱敏标记。保存时，未改变的标记会在内存中恢复磁盘值；用户不能用明文替换它。这让已有的旧版明文密钥在编辑其他字段时保持不变且不暴露于 UI。新密钥引用必须以 `!` 开头，并应指向 Keychain 命令；请通过应用保存服务商以迁移旧值。
